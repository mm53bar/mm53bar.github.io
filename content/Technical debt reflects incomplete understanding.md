Technical debt isn't just messy or untested code - it’s the gap between the understanding you have today when you write the code and the deeper understanding you'll have in six months. Looked at this way, technical debt is inevitable.

Stable teams can mitigate this because they have seen more context for the problem. They also have time to revisit past decisions with deeper context ([[Stable teams compound knowledge into market advantage]]).

**Connections**:

- Reinforces [[Learning as a business strategy]]: Debt accrues fastest where learning is deferred.
- AI corollary: [[AI-generated code risks hidden debt]] when applied to poorly understood problems.
